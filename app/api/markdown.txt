Markdown 是一种轻量级标记语言，由约翰·格鲁伯（John Gruber）在2004年创建。它的设计目标是让人们能够用易读易写的纯文本格式编写文档，并可方便地转换为结构化的 HTML 网页。Markdown 广泛用于技术文档、笔记、论坛帖子、静态网站生成器等场景。其核心理念是“让文档在纯文本状态下就具有较好的可读性”，同时通过简单的标记符号实现排版功能。

Markdown 的主要特点

· 语法简单：只有少量符号，学习成本低。
· 纯文本：任何文本编辑器都能编写，无需专门软件。
· 兼容性强：可转换为 HTML、PDF、Word 等多种格式。
· 平台支持广泛：GitHub、知乎、简书、Typora 等大量平台和工具都支持 Markdown。

常用 Markdown 语法
> 请注意，以下语法均是英文符号
请勿使用中文，全角符号

1. 标题

使用 # 表示标题级别，# 越多标题层级越低。

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

2. 段落与换行

· 段落之间用空行分隔。
· 行尾加两个空格再回车可强制换行。

3. 强调

· 斜体：文字两侧各加一个星号 * 或下划线 _（如 *斜体*）。
· 粗体：文字两侧各加两个星号 ** 或下划线 __（如 **粗体**）。
· 粗斜体：两侧各加三个星号 ***。

4. 列表

无序列表

使用 -、+ 或 * 开头：

```markdown
- 项目1
- 项目2
  - 嵌套项目（缩进2或4空格）
```

有序列表

使用数字加点 .：

```markdown
1. 第一项
2. 第二项
   1. 嵌套有序列表
```

5. 链接

· 行内式：[显示文本](https://example.com "可选标题")
· 参考式：[显示文本][id]，然后在文档末尾定义 [id]: https://example.com
· 自动链接：<https://example.com> 或 <email@example.com>

6. 图片

与链接类似，前面加叹号 !：

```markdown
![替代文字](图片地址 "可选标题")
```

7. 代码

· 行内代码：用反引号  `code`  包裹。
· 代码块：用三个反引号   包裹，并可指定语言（如  ```python ）实现语法高亮。
  ```markdown
  ```python
  def hello():
      print("Hello, Markdown!")
  ```
  ```

8. 引用

使用 > 开始引用块：

```markdown
> 这是一段引用。
> 可以换行。
>> 也可以嵌套引用。
```

9. 水平分割线

三个或以上的 -、* 或 _ 单独成行：

```markdown
---
***
___
```

10. 表格

使用管道符 | 和短横线 - 分隔单元格，第二行定义对齐方式：

```markdown
| 左对齐 | 居中对齐 | 右对齐 |
| :----- | :------: | -----: |
| 单元格 | 单元格   | 单元格 |
```

11. 任务列表

某些扩展支持（如 GitHub Flavored Markdown）：

```markdown
- [x] 已完成任务
- [ ] 未完成任务
```

12. 转义字符

若想显示原本有特殊含义的符号，前面加反斜杠 \，如 \* 显示星号。
